# Security Policy

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability or leaked credential. Use GitHub's private vulnerability reporting feature when the repository enables it. Until then, contact the maintainer privately through the email address listed in the repository's GitHub profile.

Include a clear impact description, affected version or commit, reproduction steps, and suggested mitigation. Do not include active credentials in the report.

## Leaked Credentials

Treat a credential committed to Git history as exposed. Revoke or rotate it immediately. Removing it from the current tree or rewriting history does not make the credential safe to continue using.
